# FM-SC-001: Create New Form Successfully

## Scenario Title
Create a new form with basic configuration and verify it is saved successfully.

## Prerequisites
- User is logged into the CRM application
- User has permissions to create forms
- User is on the Forms dashboard or Forms listing page

## Test Steps

1. Navigate to the Forms section from the main navigation menu
2. Click on the "Create New Form" or "+ New Form" button
3. Enter a unique form name (e.g., "Customer Feedback Form")
4. Enter a form description (e.g., "Collect customer feedback on product experience")
5. Select a form category or type if applicable (e.g., "Customer Service")
6. Set form status to "Draft" or "Active" as appropriate
7. Click the "Save" or "Create Form" button
8. Observe the success notification or message
9. Verify you are redirected to the form details or form builder page
10. Confirm the form appears in the Forms listing with the correct name and details

## Expected Result
- The form is created successfully
- A success message is displayed (e.g., "Form created successfully")
- The new form appears in the Forms listing
- Form details match the entered information
- User is able to access the newly created form for further configuration

## Test Data
- **Form Name**: "Customer Feedback Form"
- **Description**: "Collect customer feedback on product experience"
- **Category**: "Customer Service"
- **Status**: "Draft"

## Notes
- Verify that form names have uniqueness validation
- Check that required fields are properly enforced
- Test with various form name lengths and special characters if needed
